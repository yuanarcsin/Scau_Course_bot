# 华农选课助手 — 使用指南

## 一、安装

```bash
pip install httpx beautifulsoup4
```

## 二、配置

编辑 `course_bot/config.py`，修改以下内容：

```python
# 必填
student_id: str = "<你的学号>"
password: str = "<你的密码>"

# 目标课程（顺序即优先级，批量提交时同时发送）
target_courses: list = field(default_factory=lambda: [
    {"jxbbh": "<教学班编号>", "kklxdm": "01"},
])

# 选课窗口时间（一般无需修改，脚本自动从服务器获取）
window_open: str = "2026-06-23 12:30:00"

# 保留浏览器登录状态（默认 True，不踢浏览器）
preserve_browser_session: bool = True
```

### 如何获取 jxbbh

1. 浏览器登录教务系统 → 选课页面
2. 课程名旁显示的教学班编号即 `jxbbh`，格式 `学年学期-课程号-课序号`
3. `kklxdm`：01=专业选修课  06=板块课（体育/英语） 10=通识选修课

## 三、前置步骤：添加选课意向

运行脚本前，**必须在浏览器中将目标课程添加到"我的选课意向"**：
1. 浏览器登录 → 选课页面 → 找到目标课程
2. 点击"添加"按钮，加入选课意向
3. 确认选课意向列表中已有所需课程

脚本自动从选课意向获取 `xkgwcb_id`，无需手动配置。

## 四、运行

```bash
python run.py                    # 完整流程（登录 → 等待窗口 → 批量提交 → 校验 → 持续重试）
python run.py --now              # 跳过窗口等待，立即提交（窗口未开则自动转入等待）
python run.py --hunt             # 主流程完成后持续轮询未选中的课程
python run.py --window "12:30:00"  # 覆盖窗口时间
```

## 五、完整流程

```
登录(RSA加密) → 匹配课程(选课意向) → 静默保活(HEAD/120s)
→ 窗口到 → 批量SubmitCart(一次POST) → 校验(查询已选列表)
→ 失败课程持续重试 → 全部成功/终态退出
```

| 阶段 | 说明 |
| --- | --- |
| 登录 | RSA 加密，默认不踢浏览器 session |
| 匹配 | 查询选课意向 → 按 jxbbh 匹配 → 获取 xkgwcb_id |
| 等待 | 实时倒计时 + HEAD 保活(默认 120s) + 窗口前 NAT 刷新 |
| 提交 | 窗口开启瞬间，**一次 POST 批量提交**所有课程 IDs |
| 校验 | 查询已选课程列表，确认课程确实入选课表 |
| 重试 | 失败课程持续循环，遇到窗口未开/会话过期自动恢复 |

## 六、结果分类（全覆盖）

服务器每次返回由 `classify()` 归类为六种结果之一：

| 类型 | 含义 | 后续动作 | 触发条件 |
| --- | --- | --- | --- |
| **success** | 课程已选中 | 退出重试 | `flag=1` / msg含"成功" / msg含"已选/已修/重复" |
| **full** | 终态：容量已满 | 退出重试 | msg含"满/容量/名额/余量不足" |
| **stop** | 终态：业务拒绝 | 退出重试 | msg含"冲突/限制/培养方案/不符合/不允许"等 |
| **window** | 窗口未开 | 转入窗口等待 | msg含"未开始/不在选课时间/暂未开始" |
| **auth** | 会话过期 | 自动重新登录 | HTTP 401/403 或 msg含"登录过期/失效" |
| **retry** | 临时故障 | 等待后重试 | 超时/服务器繁忙/频率过高/未知响应 |

### 重试策略

```
success/full/stop → 不再重试（终态）
window           → 所有 pending 课程都是 window → 转入 _wait_window()
auth             → 任一课程 auth → _relogin()
retry            → 等待 0.05s → 重新批量提交
```

### 校验机制

SubmitCart 返回 success 后，立即查询"已选课程列表"确认课程确实出现。若不存在 → 退回 retry。

## 八、错误码速查

| 错误码 | 含义 | 类型 | 处理 |
| --- | --- | --- | --- |
| E1001 | CSRF 令牌未找到 | - | 检查 base_url |
| E1002 | RSA 公钥获取失败 | - | 服务器可能维护中 |
| E1003 | 登录失败 | - | 检查学号密码 |
| E1101 | 会话已过期 | auth | 自动重新登录 |
| E1203 | 未找到目标课程 | - | 检查 jxbbh 或添加到选课意向 |
| E1401 | 请求超时 | retry | 自动重试 |
| E1502 | 登录失败次数过多 | - | 等锁定期过 |

## 十、速度参数

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `snipe_timeout` | 0.5s | 提交请求超时 |
| `submit_retry_delay` | 0.05s | 重试间隔 |
| `request_timeout` | 2.0s | 一般请求超时 |
| `session_keepalive` | 120s | 等待期间保活间隔 |
| `nat_refresh_seconds` | 5.0s | 窗口前 NAT 刷新时间 |

批量提交只需 1 次 POST（vs 旧版 N 次），共享单个 httpx 连接池。
